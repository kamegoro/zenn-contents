## TL;DR

- **tobira.nvim** は Vim の操作を**黙って観察**して改善できそうなタイミングに「こっちのほうが良かったよ」とそっと教えてくれる Neovim プラグイン
- キーを封じない・割り込まない。操作はいつでも**自分のペース**でできる
- 136 コマンドの学習グラフを持ち、**自分のレベルに合った提案だけ**が届く
- `j` 5 回連打・`dw` → `i` → 書き直し・`f` の繰り返しなど、実際の癖をリアルタイムで検出する

## なぜ作ったのか

Vim を使い始めて数年経ちますが、こういう状態になっていました。

> 「今ある引き出しで何とかしている。新しいコマンドを積極的に覚えようとはしていない」

かといって、操作を強引に矯正されるほど熱量を注ぎたいわけでもない。ゆっくり自分のペースで気づけるものがほしくて、なかったので作りました。同じような温度感の人、いないですかね。

## tobira.nvim の動き

### 提案フロー

`vim.on_key()` でキー操作をパッシブに記録し続けています。1.5 秒間キー入力が止まると、直前の操作を解析してパターンを検出します。改善余地があれば右下にフローティングウィンドウで提案が出ます。

![tobira の提案フロー — f を繰り返した直後に ; が提案される](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-suggest.gif)

提案は 10 秒後に自動で消えます。作業の邪魔にはなりません。提案メッセージは日本語・英語を切り替えられます（`lang = 'ja'`）。

### 検出しているパターン

| やってしまいがちな操作 | tobira の提案 |
|---|---|
| `fa` → `fa` と同じ行で繰り返す | `;` — 直前の f/t を繰り返す |
| `dw` → `i` → 書き直す | `cw` / `ciw` |
| `x` を 3 回以上連続 | `{n}x` — カウントプレフィックス |
| `j` を 5 回以上連続 | `{n}j` / `<C-d>` |
| `0` → `w` で行頭に移動 | `^` — 最初の非空白文字 |
| 検索後に `n` を 4 回以上 | `cgn` — 次のマッチを変更 |

### ガイドパネル

`:TobiraGuide` でチートシートを開けます。習得済みコマンドには `✓` が付きます。次のステップは `→` で示されます。

![TobiraGuide — チートシートパネル](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-guide.gif)

### スキルツリー

`:TobiraProgress` で現在のレベルと学習グラフを確認できます。`x` キーで特定の提案を封印・解除もできます。

![TobiraProgress — スキルツリーとレベル表示](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-progress.gif)

レベルは `novice / beginner / intermediate / advanced` の 4 段階で、**自分の操作ログから自動判定**されます。レベルに応じて提案されるコマンドが絞られるので、beginner のうちに `cgn` の提案は届きません。

## インストール

**lazy.nvim**

```lua
{
  "kamegoro/tobira.nvim",
  event = "VeryLazy",
  opts = {},
}
```

設定オプションはすべてオプションで、`opts = {}` だけでも動きます。

```lua
require("tobira").setup({
  lang                = 'ja',    -- 'en' | 'ja'（提案メッセージの言語）
  idle_delay          = 1500,    -- 提案を出すまでの待機時間（ms）
  idle_suggestions    = true,    -- idle トリガーの提案を有効にするか
  suggestion_cooldown = 300,     -- 自動提案の最小インターバル（秒）
  max_shown           = 3,       -- 同じコマンドを提案する最大回数/セッション
})
```

**コマンド一覧**

| コマンド | 説明 |
|---|---|
| `:Tobira` | 今すぐ次の提案を表示（クールダウン無視） |
| `:TobiraGuide` | チートシートパネルを開閉 |
| `:TobiraProgress` | スキルツリーとレベルを表示 |
| `:TobiraStats` | コマンド使用分布と効率ギャップを表示 |
| `:TobiraReset` | 学習データをリセット |

## tobira.nvim の立ち位置

既存の Vim 学習補助プラグインはアプローチが大きく 3 種類あります。

| タイプ | 動き |
|---|---|
| 矯正型 | 非効率な操作をブロックして強制的に直す |
| 常時表示型 | 使えるコマンドを画面に出し続ける |
| **パッシブ提案型（tobira）** | 操作を観察してタイミングよく一言だけ言う |

## 正直なところ

自分自身もまだ Vim をキャッチアップしている途中なので、「このプラグインで上達した」と言い切れるほど使い込んでいるわけではありません。

自分で使いたいものを自分で作るのは単純に楽しいです。

課題もあります。パターン検出はまだ荒削りで、意図した操作に提案が届くこともあります。提案内容のカスタマイズも現時点では未対応です。「作った本人が一番のベータテスター」という状態で、使いながら改善していくつもりです。
