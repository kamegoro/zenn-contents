## TL;DR

- **tobira.nvim** は Vim の操作を**黙って観察**して、改善できそうなタイミングに「こっちのほうが良かったよ」とそっと教えてくれる Neovim プラグイン
- キーを封じない・割り込まない。操作はいつでも**自分のペース**でできる
- 145 コマンドの学習グラフを持ち、**自分のレベルに合った提案だけ**が届く
- `j` 5 回連打・`dw` → `i` → 書き直し・`f` の繰り返しなど、実際の癖をリアルタイムで検出する

## なぜ作ったのか

Vim を使い始めて数年経ちますが、こういう状態になっていました。

> 「今ある引き出しで何とかしている。新しいコマンドを積極的に覚えようとはしていない」

かといって、操作を強引に矯正されるほど熱量を注ぎたいわけでもない。ゆっくり自分のペースで気づけるものがほしくて、なかったので作りました。

## tobira.nvim の動き

### 提案フロー

`vim.on_key()` でキー操作をパッシブに記録し続けています。1.5 秒間キー入力が止まると、直前の操作を解析してパターンを検出します。改善余地があれば右下にフローティングウィンドウで提案が出ます。

![tobira の提案フロー — f を繰り返した直後に ; が提案される](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-suggest.gif)

提案は 10 秒後に自動で消えます。作業の邪魔にはなりません。提案メッセージは日本語・英語を切り替えられます（`lang = 'ja'`）。

### 検出しているパターン（34 種類）

**移動系**

| やってしまいがちな操作 | tobira の提案 |
|---|---|
| `fa` → `fa` と同じ行で繰り返す | `;` — 直前の f/t/F/T を繰り返す |
| `j` を 5 回以上連続 | `{n}j` — カウントプレフィックス |
| `j` を 10 回以上連続 | `}` — 段落単位で移動 |
| `l` を 5 回以上連続 | `w` — 単語単位で移動 |
| `0` → `w` で行頭移動 | `^` — 最初の非空白文字へ |
| `0` / `^` → `i` | `I` — 行頭から挿入 |
| `$` → `a` | `A` — 行末に追記 |
| `k`（1 回）→ `o` | `O` — 上に新しい行を開く |
| 検索後に `n` を 4 回以上 | `cgn` — 次のマッチを変更 |

**編集系**

| やってしまいがちな操作 | tobira の提案 |
|---|---|
| `dw` → `i` → 書き直す | `cw` — 削除 + 挿入を 1 コマンドで |
| `c$` | `C` — 行末まで変更（短縮形） |
| `d$` | `D` — 行末まで削除（短縮形） |
| `yy` → `p` | `yyp` — 行の複製 |
| `dd` → `p` | `ddp` — 行を下と入れ替え |
| `dd` → `i` / `a` / `o` | `cc` — 行全体を変更 |
| `dd` を 3 回以上 | `{n}dd` — N 行まとめて削除 |
| `x` → `i` / `a` / `o` | `s` — 1 文字削除 + 挿入 |
| `x` を 3 回以上 | `{n}x` — N 文字まとめて削除 |
| `u` を 3 回以上 | `<C-r>` — redo でやり直す |
| `r{x}` を 3 回以上 | `R` — Replace モードで連続置換 |
| `>>` を 3 回以上 | `{n}>>` — N 行まとめてインデント |
| `v` `i` `{obj}` `c/d/y` | `ciw` / `di"` など — 直接 text-object を使う |

この他にも `k × 10 → {`・`p × 3 → {n}p`・`.× 3 → {n}.`・`J × 3 → {n}J` など 34 種類を検出します。完全な一覧は `:help tobira-patterns` で確認できます。

### 採用を「行動」で検出する

提案した後、tobira はそのコマンドを実際に使ったかどうかをパッシブに監視しています。ボタンを押す必要はなく、提案されたコマンドを使えば自動的に「採用」とみなします。

採用されたコマンドは累計 100 回使用を超えると「習得」扱いになり、以降は提案されません。

### ガイドパネル

`:TobiraGuide` でレベルに応じたチートシートを開けます。

![TobiraGuide — チートシートパネル](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-guide.gif)

### スキルツリー

`:TobiraProgress` で現在のレベルと学習グラフを確認できます。

![TobiraProgress — スキルツリーとレベル表示](https://raw.githubusercontent.com/kamegoro/tobira.nvim/main/docs/demo-progress.gif)

習熟度は使用回数に応じて 4 段階で表示されます。

| 表示 | 意味 |
|---|---|
| （なし） | 未使用 |
| `☆` | 1 回以上使用 |
| `★` | 100 回以上 |
| `★★` | 1000 回以上 |
| `★★★` | 5000 回以上 |

`x` キーで特定の提案を封印・解除、`p` キーでピン留め（チートシート上部に常時表示）ができます。

レベルは `novice / beginner / intermediate / advanced` の 4 段階で、**自分の操作ログから自動判定**されます。レベルに応じて提案されるコマンドが絞られるので、beginner のうちに `cgn` の提案は届きません。

## インストール

**lazy.nvim**

```lua
{
  "kamegoro/tobira.nvim",
  event = "VeryLazy",
  opts = {},
}
```

設定オプションはすべて省略可能で、`opts = {}` だけでも動きます。

```lua
require("tobira").setup({
  lang                = 'ja',    -- 'en' | 'ja'
  idle_delay          = 1500,    -- 提案を出すまでの待機時間（ms）
  idle_suggestions    = true,    -- idle トリガーの提案を有効にするか
  suggestion_cooldown = 300,     -- 自動提案の最小インターバル（秒）
  max_shown           = 2,       -- 同じコマンドを提案する最大回数/セッション
})
```

**コマンド一覧**

| コマンド | 説明 |
|---|---|
| `:Tobira` | 今すぐ次の提案を表示（クールダウン無視） |
| `:TobiraGuide` | チートシートパネルを開閉 |
| `:TobiraProgress` | スキルツリーとレベルを表示 |
| `:TobiraStats` | コマンド使用分布と効率ギャップを表示 |
| `:TobiraReset` | 学習データをリセット |

`:help tobira` で全オプション・コマンドのドキュメントを参照できます。

## tobira.nvim の立ち位置

既存の Vim 学習補助プラグインはアプローチが大きく 3 種類あります。

| タイプ | 動き |
|---|---|
| 矯正型（hardtime.nvim など） | 非効率な操作をブロックして強制的に直す |
| 常時表示型 | 使えるコマンドを画面に出し続ける |
| **パッシブ提案型（tobira）** | 操作を観察してタイミングよく一言だけ言う |

hardtime.nvim と tobira.nvim は競合しません。「hardtime が悪い癖をブロックし、tobira が次の扉を開ける」という使い方ができます。

## 技術的なポイント

### vim.on_key() だけで全操作を把握する

tobira は `vim.on_key()` 以外のキー追跡 API を一切使いません。`TextYankPost` などの autocmd を使うと操作の一部しかカバーできず、ユーザーのキーマッピングを書き換えるリスクもあります。`vim.on_key()` は全キーストロークを受け取るため、演算子文法を自前でパースすれば意味レベルの検出が可能です。

Neovim 0.10 から `vim.on_key()` のコールバックに `typed`（ユーザーが実際に打鍵した物理キー）引数が追加されました。`D` を押すと Neovim 内部で `d` と `$` が展開されますが、`typed` を見ることで「ユーザーが打ったのは `D` 1 キー」と正確に判断できます。0.9 以前でもフォールバックで動作します。

### 状態機械でパターンを検出

`d3w`・`diw`・`daw` はどれも「削除 + motion」なので同じパターンとして扱いたい。また `gj`・`zz` などの 2 文字コマンドは `j` や `z` 単体と区別したい。これらを `vim.on_key()` 1 発 1 発から意味として拾うために、`patterns.lua` に小さな状態機械を実装しています。

```
d / c が来る → pending_op = 'd' | 'c'

pending_op 中:
  [1-9]             → カウント数字（d3w 対応）
  dd / dj / dk      → 行削除
  i / a の後        → text-object 待ち（diw / daw 対応）
  w / e / $ など    → charwise（dw として扱う）
  <Esc>             → キャンセル
```
